## Примечание

В текущей версии данные JSON-профили рассматриваются как внутренние конфигурации системы. Пользователь работает со штатными профилями через TUI и не использует свободное конструирование среды.

---

## Формат профилей

Профили в TUIopenMath описывают состав вычислительной среды.  
Пользователь может менять профиль, добавлять новый или редактировать существующий, не изменяя код программы, если формат профиля соблюдён.

Профиль задаёт:

- имя профиля;
- описание;
- Python-пакеты;
- Julia-пакеты;
- системные компоненты;
- ядра Jupyter;
- параметры рабочего пространства.

---

## Общая структура профиля

Ниже приведён пример профиля в формате JSON:

```json
{
  "name": "math",
  "description": "Математический профиль для символьных и численных вычислений",
  "python": [
    "numpy",
    "scipy",
    "sympy",
    "matplotlib",
    "jupyterlab"
  ],
  "julia": [
    "IJulia",
    "Plots",
    "Symbolics"
  ],
  "system": [
    "octave"
  ],
  "kernels": [
    "python",
    "julia",
    "bash",
    "octave"
  ],
  "workspace": {
    "readme_title": "Math Workspace",
    "starter_notebook": true
  }
}
```

---

## Поля профиля

### `name`

Имя профиля.

#### Разрешено:
- только короткое имя;
- латинские буквы;
- цифры;
- символы `-` и `_`, если это разрешено логикой проекта.

#### Рекомендуется:
- использовать простые имена:
  - `base`
  - `math`
  - `ml`
  - `symbolic`

#### Нежелательно:
- пробелы;
- кавычки;
- спецсимволы shell;
- слишком длинные названия.

#### Пример:
```json
"name": "math"
```

---

### `description`

Текстовое описание профиля.

#### Можно:
- писать понятное описание назначения профиля;
- использовать обычный текст на русском или английском.

#### Пример:
```json
"description": "Профиль для численных и символьных вычислений"
```

---

### `python`

Список Python-пакетов, которые должны быть установлены в профильное virtualenv.

#### Можно указывать:
- обычные имена пакетов из PyPI;
- пакеты, которые устанавливаются через `pip`.

#### Примеры корректных значений:
- `numpy`
- `scipy`
- `sympy`
- `matplotlib`
- `pandas`
- `scikit-learn`
- `jupyterlab`

#### Нельзя:
- указывать shell-команды;
- писать `pip install ...`;
- вставлять флаги командной строки;
- использовать небезопасные строки.

#### Правильно:
```json
"python": [
  "numpy",
  "scipy",
  "sympy"
]
```

#### Неправильно:
```json
"python": [
  "pip install numpy",
  "rm -rf /",
  "numpy && evil"
]
```

---

### `julia`

Список Julia-пакетов, которые должны быть установлены в Julia-среду.

#### Можно указывать:
- только имена пакетов Julia.

#### Примеры корректных значений:
- `IJulia`
- `Plots`
- `Symbolics`

#### Нельзя:
- писать команды Julia вручную;
- использовать произвольный код;
- вставлять shell-команды.

#### Правильно:
```json
"julia": [
  "IJulia",
  "Plots",
  "Symbolics"
]
```

#### Неправильно:
```json
"julia": [
  "using Pkg; Pkg.add(\"IJulia\")",
  "run(`rm -rf /`)"
]
```

---

### `system`

Список системных компонентов, которые нужны профилю.

Это не локальные Python-пакеты, а внешние инструменты, устанавливаемые через систему.

#### Примеры допустимых значений:
- `octave`
- `scilab`

#### Важно:
Поле `system` должно содержать **только имена известных поддерживаемых системных компонентов**, а не произвольные команды.

#### Правильно:
```json
"system": [
  "octave"
]
```

#### Неправильно:
```json
"system": [
  "sudo apt-get install octave",
  "bash -c 'something'"
]
```

---

### `kernels`

Список ядер Jupyter, которые нужно зарегистрировать для профиля.

#### Поддерживаемые значения:
- `python`
- `julia`
- `bash`
- `octave`
- `scilab`

#### Можно:
- включать только нужные ядра;
- задавать несколько ядер для одного профиля.

#### Пример:
```json
"kernels": [
  "python",
  "julia",
  "bash",
  "octave"
]
```

#### Нельзя:
- указывать неизвестные ядра, если они не поддерживаются системой;
- писать произвольные строки.

#### Неправильно:
```json
"kernels": [
  "python",
  "php",
  "my-super-kernel"
]
```

---

### `workspace`

Объект с параметрами рабочего пространства профиля.

#### Возможные поля:
- `readme_title` — заголовок README для workspace;
- `starter_notebook` — нужно ли создавать стартовый notebook.

#### Пример:
```json
"workspace": {
  "readme_title": "Math Workspace",
  "starter_notebook": true
}
```

---

## Что пользователь может менять безопасно

Пользователь может безопасно:

- менять `description`;
- добавлять и убирать Python-пакеты;
- добавлять и убирать Julia-пакеты;
- включать или отключать `octave` и `scilab` в `system`;
- задавать набор поддерживаемых kernels;
- менять название workspace и включать стартовый notebook.

---

## Что пользователь не должен делать

Пользователь **не должен**:

- вставлять shell-команды в поля профиля;
- писать `pip install ...` вместо имени пакета;
- вставлять произвольный код в поле `julia`;
- использовать неподдерживаемые имена kernels;
- использовать опасные символы в `name`;
- пытаться описывать логику установки в самом профиле.

Профиль — это **данные**, а не сценарий.

## Рекомендации по составлению профиля

Рекомендуется:

- делать профиль компактным;
- включать только действительно нужные пакеты;
- разделять профили по назначению;
- не смешивать всё в один универсальный профиль;
- использовать понятные имена;
- не добавлять неподдерживаемые компоненты.

---

## Логика расширения

Сильная сторона системы в том, что пользователь может:

- редактировать существующий профиль;
- создавать новый профиль;
- менять состав среды через конфигурацию профиля;
- не переписывать для этого прикладной код программы.

Именно это делает профили удобным способом управления средой.
