# Патч обновления №3 для TUIopenMath / SysOpenMath

## Назначение патча

Данный патч продолжает развитие архитектуры SysOpenMath / TUIopenMath как управляемой вычислительной платформы.

Основной целью патча является:

1. введение единой модели состояния компонентов;
2. разделение платформенной и профильной диагностики;
3. реализация проверки версий компонентов;
4. улучшение логики запуска JupyterLab;
5. стабилизация поведения kernels;
6. улучшение пользовательского интерфейса и splash-screen.

Патч ориентирован на дальнейшее приближение платформы к модели штатной математической вычислительной среды, в которой пользователь взаимодействует не с набором отдельных пакетов, а с готовой управляемой системой.

---

# Основные изменения

## 1. Введение общей модели InstallStatus

В предыдущих версиях проверки компонентов выполнялись частично независимо друг от друга.

В рамках данного патча вводится единая модель состояния компонентов:

```c
typedef enum InstallStatus
{
    INSTALL_STATUS_READY,
    INSTALL_STATUS_BROKEN,
    INSTALL_STATUS_MISMATCH,
    INSTALL_STATUS_NOT_INSTALLED,
    INSTALL_STATUS_SKIP
} InstallStatus;
```

Данная модель используется для:

- Python-пакетов;
- Julia-пакетов;
- системных компонентов;
- kernels;
- JupyterLab;
- runtime-компонентов.

---

## 2. Разделение платформенной и профильной диагностики

Ранее диагностика частично смешивала:

- глобальные проверки платформы;
- проверки активного профиля.

Это приводило к ложным ошибкам и избыточной диагностике.

После патча проверки разделяются на два уровня:

### Платформенная диагностика

Проверяет:

- Python runtime;
- Jupyter runtime;
- наличие bash;
- базовую структуру каталогов;
- системные зависимости платформы.

### Профильная диагностика

Проверяет исключительно то, что входит в активный профиль:

- Python dependencies;
- Julia dependencies;
- kernels;
- системные компоненты профиля.

Таким образом:

```text
platform checks != profile checks
```

---

## 3. Реализация проверки версий компонентов

Патч добавляет систему version checks.

Теперь проверяются версии:

- Julia;
- JupyterLab;
- Octave;
- Scilab;
- Python runtime.

Примеры:

```text
[OK] julia version 1.10.10
[OK] JupyterLab 4.5.7
[OK] Octave 9.x
```

Если версия несовместима:

```text
[MISMATCH] unsupported Julia version
```

---

## 4. Развитие логики READY / BROKEN / MISMATCH / SKIP

Проверка состояния среды теперь строится вокруг общей readiness-модели.

### READY

Компонент полностью готов к использованию.

### BROKEN

Компонент установлен, но работает некорректно.

Примеры:

- повреждён kernel;
- broken venv;
- отсутствуют runtime-файлы.

### MISMATCH

Обнаружена несовместимая версия.

### SKIP

Компонент не относится к выбранному профилю.

Пример:

```text
[SKIP] scilab kernel not required
```

---

## 5. Улучшение проверки готовности профиля

Профиль теперь рассматривается как единая вычислительная среда.

Система определяет:

- READY;
- PARTIALLY READY;
- NOT READY.

Профиль считается READY только если:

- существует environment;
- установлены зависимости;
- зарегистрированы kernels;
- JupyterLab доступен;
- отсутствуют critical errors.

---

## 6. Улучшение запуска JupyterLab

В предыдущих версиях запуск JupyterLab мог:

- завершаться сигналом;
- не отображать URL;
- открывать временный runtime HTML;
- создавать несколько процессов.

Патч изменяет логику запуска.

Теперь:

- JupyterLab запускается в background;
- URL извлекается из логов;
- ссылка выводится пользователю;
- предотвращается повторный запуск.

Пример:

```text
[INFO] JupyterLab started
[INFO] URL: http://localhost:8888/lab?token=...
```

---

## 7. Улучшение работы kernels

Патч стабилизирует:

- Python kernel;
- Julia kernel;
- Bash kernel;
- Octave kernel;
- Scilab kernel.

Добавлены:

- проверки регистрации;
- проверки исполняемых файлов;
- проверки kernelspec;
- обработка отсутствующих kernels.

Примечание: Раньше Юпитер не видел удаление ядер внутри, при проверке в системе ядер не было, но в самом Юпитере интерфейс показывал Ядра все которые скачивались но они были пустышками - этот баг был решен! 

---

## 8. Улучшение splash-screen и TUI

В рамках патча доработан терминальный интерфейс.

Добавлены:

- splash-logo SysOpenMath;
- центрирование ASCII-art;
- ожидание нажатия клавиши;
- улучшение обработки resize;
- логирование resize events.

Также улучшено отображение логов установки и диагностики.

---

## 9. Улучшение структуры логирования

Теперь система логирует:

- запуск команд;
- статусы проверок;
- version checks;
- readiness checks;
- запуск JupyterLab;
- kernel registration.

Пример:

```text
[INFO] Checking profile symbolic
[OK] python import: sympy
[OK] julia imports
[SKIP] scilab not required
[READY] profile symbolic
```

- снижение ложных ошибок;

- повышение предсказуемости платформы.
